一个基于 tauri 的 ffmpeg 包装程序，支持以下功能：
- 允许用户将文件拖放到窗口来添加，并显示视频的基本参数
- 自动检测用户是否安装 ffmpeg（假设在 PATH 中），如果没有则提示用户安装（后续考虑添加自动下载 ffmpeg 的功能）
- 自动检测用户的操作系统以及支持的硬件加速
- 设定视频参数（编码、crf、码率等）
- 设定视频中音频轨道的处理方式（复制流、移除音频等）
- 设定画面滤镜（分辨率、长度裁剪等）

前端使用 Vue，暂时不引入组件库。

采用 **“左右分栏 + 底部状态栏”** 的布局。这种布局在处理批量任务和复杂参数时最为高效。

### 1. 总体布局方案

*   **左侧栏 (40% 宽度)：任务管理区**
    *   顶部是一个巨大的“拖放”感应区。
    *   下方是任务列表（Queue），显示文件名、原始大小、当前状态（等待中/处理中/已完成）。
*   **右侧栏 (60% 宽度)：参数配置区**
    *   采用 **Tab 标签页** 或 **垂直折叠面板 (Accordion)** 来组织参数。
    *   分为：视频设置、音频设置、高级处理（滤镜/剪裁）。
*   **底部栏 (固定高度)：执行与状态**
    *   左侧：FFmpeg 状态（如：`FFmpeg: Ready (v6.0)`）。
    *   右侧：全局进度条、预计剩余时间、开始/停止按钮。

### 2. 详细 UI 模块设计

#### A. 左侧：任务列表 (Task List)
*   **空状态：** 居中的虚线框，带有一个“+”图标和“拖入视频文件”字样。
*   **非空状态：** 紧凑的卡片式列表。
    *   每个卡片显示：文件名、分辨率、时长。
    *   状态指示：处理中显示微型进度条，成功显示绿勾，失败显示红叉。

#### B. 右侧：参数配置面板 (Config Panel)
为了不让界面显得拥挤，建议分为四个折叠块：

1.  **视频 (Video) - 默认展开**
    *   **编码器：** 两个下拉框（[格式: H264/H265/AV1] [硬件加速: CPU/NVENC/Toolbox]）。
    *   **质量控制：** 
        *   单选切换：[CRF模式] 或 [目标码率模式]。
        *   滑动条 (Slider)：CRF 值（18-28）或 码率数值输入框。
    *   **预设速度：** 极慢 -> 极快 (Preset: Slower to Ultrafast)。

2.  **音频 (Audio) - 默认展开**
    *   下拉框：[复制流 (Copy)]、[移除音频 (None)]、[AAC 压缩]、[MP3 压缩]。
    *   如果是压缩，显示码率选择（128k, 192k, 320k）。

3.  **画面与滤镜 (Advanced) - 默认折叠**
    *   **分辨率：** 下拉框（原始、1080P、720P 等）。
    *   **裁剪 (Trim)：** 两个输入框（开始时间、结束时间）。
    *   **字幕：** 拖放区（支持 .srt/.ass）和 [硬压/软封装] 切换开关。

4.  **输出设置 (Output)**
    *   输出目录选择（默认：原文件夹/out）。
    *   文件名后缀设置。

#### C. 底部：全局控制 (Global Control)
*   **左侧：** 一个小绿点，写着 "FFmpeg 6.0 | GPU Accel: On"。
*   **中间：** 总进度百分比。
*   **右侧：** 
    *   [清空列表] 按钮（次要）。
    *   [立即开始] 按钮（主要，亮色/品牌色）。
